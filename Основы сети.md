Разница между `ifconfig` и `ip a` (часть команды `ip addr`) важна для понимания современных Linux-систем.

В Linux управление сетью происходит через терминал. Две основные команды:
- **`ifconfig`** — устаревшая, но простая и популярная(пакет `net-tools`).
- **`ip addr`** (чаще `ip a`, `ip link`) — современная замена (пакет `iproute2`).

## **Команда `ifconfig`**

### **Базовое использование**

ifconfig                  # Показать все активные интерфейсы
ifconfig eth0             # Информация о конкретном интерфейсе
ifconfig -a               # Показать ВСЕ интерфейсы (включая выключенные)


Пример вывода:
eth0: flags=4163<UP,BROADCAST,RUNNING,MULTICAST>  mtu 1500
        inet 192.168.1.100  netmask 255.255.255.0  broadcast 192.168.1.255
        ether 00:1a:2b:3c:4d:5e  txqueuelen 1000  (Ethernet)

- **`UP`** — интерфейс активен.
- **`inet`** — IPv4-адрес.
- **`ether`** — MAC-адрес.
- **`mtu`** — максимальный размер пакета.

## **Команда `ip`**
### **Основные подкоманды:**
- `ip a` или `ip addr` — управление адресами.
- `ip link` — управление интерфейсами.
- `ip route` — управление маршрутами.
    
### **`ip a` — Просмотр адресов**

ip a                    # Краткая форма (аналог ifconfig)
ip addr show            # Полная форма
ip addr show eth0       # Только для eth0


Пример вывода:
2: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc fq_codel state UP group default qlen 1000
    link/ether 00:1a:2b:3c:4d:5e brd ff:ff:ff:ff:ff:ff
    inet 192.168.1.100/24 brd 192.168.1.255 scope global dynamic eth0
    inet6 fe80::21a:2bff:fe3c:4d5e/64 scope link 

- **`state UP`** — интерфейс активен.
- **`inet 192.168.1.100/24`** — IPv4 с маской /24 (255.255.255.0).
- **`inet6`** — IPv6-адрес.
- **`link/ether`** — MAC-адрес.

###  **`ip link` — Управление интерфейсами**

ip link set eth0 up             # Включить интерфейс
ip link set eth0 down           # Выключить интерфейс
ip link show                    # Показать все интерфейсы

## **Сравнение `ifconfig` и `ip`**

| **Функция**           | **ifconfig**                  | **ip**                                  |
| --------------------- | ----------------------------- | --------------------------------------- |
| Показать интерфейсы   | `ifconfig`                    | `ip a` или `ip link`                    |
| Включить/выключить    | `ifconfig eth0 up/down`       | `ip link set eth0 up/down`              |
| Назначить IP          | `ifconfig eth0 192.168.1.100` | `ip addr add 192.168.1.100/24 dev eth0` |
| Поддержка IPv6        | Нет (только IPv4)             | Да                                      |
| Управление маршрутами | Нет (нужен `route`)           | `ip route`                              |
|                       |                               |                                         |

> **Совет:** Всегда используйте `ip` в новых системах — он мощнее и поддерживает современные функции (например, пространства имён).

# Ping
`ping` — это одна из самых фундаментальных и часто используемых сетевых утилит для диагностики сетевых соединений. Ее основная цель — **проверить доступность удаленного хоста в сети** и измерить **время отклика** (задержку).

Название происходит от термина, используемого в гидролокации (сонар), где "пинг" — это звуковой импульс, посылаемый для обнаружения объектов.

### Как это работает?
`ping` использует протокол **ICMP (Internet Control Message Protocol)**. Она отправляет на целевой хост пакеты типа **ICMP Echo Request** и ожидает в ответ пакеты **ICMP Echo Reply**.
- **Отправляется:** `Echo Request` ("Ты жив?")
- **Принимается:** `Echo Reply` ("Да, жив!")

Если ответы приходят, значит хост доступен, и между вами есть сетевое соединение.
### Базовый синтаксис

ping [опции] <целевой_хост>

Где `<целевой_хост>` может быть:
- **IP-адрес** (например, `8.8.8.8`)
- **Доменное имя** (например, `google.com` или `yandex.ru`)
### Пример выполнения команды

Выполним простую команду:
ping google.com

Вывод будет выглядеть примерно так:
PING google.com (142.250.185.78) 56(84) bytes of data.
64 bytes from wb-in-f78.1e100.net (142.250.185.78): icmp_seq=1 ttl=117 time=12.3 ms
64 bytes from wb-in-f78.1e100.net (142.250.185.78): icmp_seq=2 ttl=117 time=11.8 ms
64 bytes from wb-in-f78.1e100.net (142.250.185.78): icmp_seq=3 ttl=117 time=11.9 ms
64 bytes from wb-in-f78.1e100.net (142.250.185.78): icmp_seq=4 ttl=117 time=12.1 ms
^C
--- google.com ping statistics ---
4 packets transmitted, 4 received, 0% packet loss, time 3005ms
rtt min/avg/max/mdev = 11.897/12.086/12.399/0.259 ms

### Разбор вывода:
1. **Первая строка:** Показывает, какой хост пингуется, его IP-адрес и размер отправляемых данных (по умолчанию 56 байт, которые вместе с заголовками ICMP и IP составляют 84 байта).
2. **Строки с ответами:** Для каждого полученного ответа выводится информация:
    - `64 bytes`: размер полученного пакета.
    - `from ...`: IP-адрес и иногда имя хоста, который ответил.
    - `icmp_seq=1`: порядковый номер пакета. Полезен для определения потерь.
    - `ttl=117`: Time To Live ("Время жизни"). Это счетчик, который уменьшается на каждом маршрутизаторе на пути. Когда TTL достигает 0, пакет уничтожается. Это предотвращает бесконечную циркуляцию пакетов в сети. Начальное значение TTL зависит от операционной системы.
    - `time=12.3 ms`: Время между отправкой запроса и получением ответа (RTT - Round-Trip Time). Измеряется в миллисекундах.
3. **Статистика (после нажатия `Ctrl+C`):**
    - `packets transmitted`: сколько пакетов было отправлено.
    - `received`: сколько пакетов получено в ответ.
    - `packet loss`: процент потерь пакетов. `0%` — идеально.
    - `rtt min/avg/max/mdev`: минимальное, среднее, максимальное время и стандартное отклонение (колебание, "джиттер").
### Ключевые опции (ключи)
По умолчанию `ping` работает непрерывно. Чтобы остановить его, нужно нажать `Ctrl+C`. Вот самые полезные опции для управления его поведением:

| Опция          | Описание                                                                                                                                    |
| -------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| `-c <число>`   | **Ограничить количество запросов.** Удобно для скриптов. Пример: `ping -c 4 google.com`                                                     |
| `-i <секунды>` | **Интервал** между отправкой пакетов (по умолчанию 1 секунда). Пример: `ping -i 2 google.com`                                               |
| `-s <размер>`  | **Размер пакета** (данных) в байтах. Полезно для тестирования MTU. Пример: `ping -s 1500 google.com`                                        |
| `-W <секунды>` | **Таймаут ожидания ответа.** Сколько ждать ответ на каждый пакет. Пример: `ping -W 5 google.com`                                            |
| `-f`           | **"Беспорядочная" отправка (flood).** Отправляет пакеты так быстро, как может. **Требует прав root!** Используется для стресс-тестирования. |
| `-q`           | **"Тихий" вывод.** Показывает только итоговую статистику.                                                                                   |
| `-4`           | Принудительно использовать **IPv4**.                                                                                                        |
| `-6`           | Принудительно использовать **IPv6**.                                                                                                        |
| `-D`           | Выводить время в формате **UNIX timestamp** перед каждой строкой.                                                                           |
| `-O`           | **Выводить сообщения о пропущенных пакетах.** Полезно для отладки.                                                                          |
