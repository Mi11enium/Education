## Уровни архитектуры linux
### **1. Аппаратное обеспечение (Hardware)**
Физические компоненты компьютера:
- Процессор (CPU)
- Оперативная память (RAM)
- Жёсткие диски (HDD/SSD)
- Сетевые карты (NIC)
- Устройства ввода/вывода (клавиатура, мышь, GPU)
### **2. Ядро (Kernel)**

**Сердце Linux** — это **ядро**, которое выполняет ключевые функции:

#### **Функции ядра**
1. **Управление процессами**
    - Создание, планирование и завершение процессов (`fork()`, `exec()`).
    - Межпроцессное взаимодействие (IPC).
2. **Управление памятью**
    - Виртуальная память (paging, swapping).
    - Выделение и освобождение памяти (`malloc()`, `free()`).
3. **Драйверы устройств (Device Drivers)**
    - Взаимодействие с железом через модули (`*.ko` в Linux).
    - Поддержка файловых систем, сетевых карт, USB и т. д.
4. **Файловая система (VFS — Virtual File System)**
    - Единый интерфейс для работы с разными ФС (ext4, NTFS, FAT32).
5. **Сетевой стек**
    - Реализация TCP/IP, UDP, маршрутизация.
6. **Безопасность**
    - Механизмы разграничения прав (`SELinux`, `AppArmor`).
    - Управление пользователями и группами.

**Важно**: Ядро работает в **привилегированном режиме**, тогда как пользовательские программы — в **непривилегированном**.

То есть Ядро имеет большие права в отличие от пользовательских программ


## **Как взаимодействуют компоненты:**

**Пример:** Запуск программы `firefox` через терминал:

1. Пользователь вводит `firefox` в **Bash (командная строка)**.
2. Bash ищет бинарный файл в файловой структуре Linux.
3. Программа загружается в **оперативную память**.
4. Ядро выделяет ресурсы (PID, файловые дескрипторы).
5. Firefox использует **библиотеки**.
6. Графический вывод идёт через **X11/Wayland**.


## **Загрузка Linux:**

1. **BIOS/UEFI** → загружает **GRUB** (загрузчик).
2. **GRUB** → загружает **ядро**.
3. **Ядро** → монтирует корневую Файловую систему и запускает **systemd**.
4. **systemd** → стартует службы (сеть, логин, графику).

Архитектура Linux — это **слоистая система**, где:
- **Ядро** управляет железом и ресурсами.
- **Библиотеки** предоставляют API для программ.
- **Пользовательские приложения** работают поверх.

Гибкость этой архитектуры позволяет Linux работать на чём угодно: от Raspberry Pi до суперкомпьютеров.
