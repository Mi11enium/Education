это **модульная операционная система**, построенная по принципу **монолитного ядра** (monolithic kernel) с поддержкой загружаемых модулей. Её архитектуру можно разделить на несколько ключевых слоёв, взаимодействующих друг с другом.

## Уровни архитектуры linux
### **1. Аппаратное обеспечение (Hardware)**
Физические компоненты компьютера:
- Процессор (CPU)
- Оперативная память (RAM)
- Жёсткие диски (HDD/SSD)
- Сетевые карты (NIC)
- Устройства ввода/вывода (клавиатура, мышь, GPU)
### **2. Ядро (Kernel)**

**Сердце Linux** — это **ядро**, которое выполняет ключевые функции:

#### **Функции ядра**
1. **Управление процессами**
    - Создание, планирование и завершение процессов (`fork()`, `exec()`).
    - Межпроцессное взаимодействие (IPC).
2. **Управление памятью**
    - Виртуальная память (paging, swapping).
    - Выделение и освобождение памяти (`malloc()`, `free()`).
3. **Драйверы устройств (Device Drivers)**
    - Взаимодействие с железом через модули (`*.ko` в Linux).
    - Поддержка файловых систем, сетевых карт, USB и т. д.
4. **Файловая система (VFS — Virtual File System)**
    - Единый интерфейс для работы с разными ФС (ext4, NTFS, FAT32).
5. **Сетевой стек**
    - Реализация TCP/IP, UDP, маршрутизация.
6. **Безопасность**
    - Механизмы разграничения прав (`SELinux`, `AppArmor`).
    - Управление пользователями и группами.

**Важно**: Ядро работает в **привилегированном режиме**, тогда как пользовательские программы — в **непривилегированном**.

То есть Ядро имеет большие права в отличие от пользовательских программ


## **Как взаимодействуют компоненты:**

**Пример:** Запуск программы `firefox` через терминал:

1. Пользователь вводит `firefox` в **Bash (командная строка)**.
2. Bash ищет бинарный файл в файловой структуре Linux.
3. Программа загружается в **оперативную память**.
4. Ядро выделяет ресурсы (PID, файловые дескрипторы).
5. Firefox использует **библиотеки**.
6. Графический вывод идёт через **X11/Wayland**.


## **Загрузка Linux:**

1. **BIOS/UEFI** → загружает **GRUB** (загрузчик).
2. **GRUB** → загружает **ядро**.
3. **Ядро** → монтирует корневую Файловую систему и запускает **systemd**.
4. **systemd** → стартует службы (сеть, логин, графику).

Архитектура Linux — это **слоистая система**, где:
- **Ядро** управляет железом и ресурсами.
- **Библиотеки** предоставляют API для программ.
- **Пользовательские приложения** работают поверх.

Гибкость этой архитектуры позволяет Linux работать на чём угодно: от Raspberry Pi до суперкомпьютеров.


## **Файловая структура Linux**

В Linux всё это — **файлы** (даже устройства и процессы!). Файловая система организована по стандарту **FHS** (Filesystem Hierarchy Standard).

Основные каталоги:

## **Корень (`/`)**

Главный каталог, содержащий все остальные.

## **Основные системные каталоги**

|              |                                                                            |                                          |
| ------------ | -------------------------------------------------------------------------- | ---------------------------------------- |
| **`/bin`**   | Базовые исполняемые файлы (команды) для всех пользователей.                | `ls`, `cp`, `bash`                       |
| **`/sbin`**  | Системные утилиты для администрирования (требуют `sudo`).                  | `fdisk`, `iptables`, `init`              |
| **`/lib`**   | Библиотеки для программ из `/bin` и `/sbin`.                               | `libc.so` (библиотека языка C)           |
| **`/usr`**   | Вторичная иерархия для пользовательских программ (аналог `Program Files`). | `/usr/bin`, `/usr/lib`, `/usr/share`     |
| **`/etc`**   | Конфигурационные файлы системы и программ.                                 | `/etc/passwd`, `/etc/network/interfaces` |
| **`/var`**   | Переменные данные (логи, кэш, почта).                                      | `/var/log`, `/var/www` (веб-серверы)     |
| **`/tmp`**   | Временные файлы (очищаются после перезагрузки).                            | —                                        |
| **`/boot`**  | Файлы загрузчика (ядро, initramfs, GRUB).                                  | `vmlinuz-5.15.0-76-generic`              |
| **`/dev`**   | Файлы устройств (жёсткие диски, USB, виртуальные терминалы).               | `/dev/sda`, `/dev/tty1`                  |
| **`/proc`**  | Виртуальная ФС с информацией о процессах и системе.                        | `/proc/cpuinfo`, `/proc/meminfo`         |
| **`/sys`**   | Информация о ядре и устройствах (на лету).                                 | —                                        |
| **`/home`**  | Домашние каталоги пользователей.                                           | `/home/alice`, `/home/bob`               |
| **`/root`**  | Домашний каталог суперпользователя (не путать с `/`!).                     | —                                        |
| **`/mnt`**   | Временные точки монтирования (например, флешек).                           | —                                        |
| **`/media`** | Автомонтирование устройств (CD/DVD, USB).                                  | `/media/usb-drive`                       |
| **`/opt`**   | Дополнительное ПО (часто проприетарное: Google Chrome, MATLAB).            | `/opt/google/chrome`                     |


## **Особенности**
1. **Нет дисков `C:\`, `D:\`**
    - В Linux разделы монтируются в каталоги (например, `/mnt/data`).
    - Жёсткий диск `/dev/sda1` может быть смонтирован в `/home`.
2. **Символьные ссылки**
    - `/bin` → `/usr/bin` (в современных дистрибутивах).
    - `/lib` → `/usr/lib`.
3. **Все настройки — в `/etc`**
    - Конфиги программ лежат здесь, а не в реестре (как в Windows).
4. **Логи — в `/var/log`**
    - `syslog`, `auth.log`, `nginx/error.log`.
