# **Что Такое Дистрибутив Linux:**

Дистрибутив Linux это не просто "версия" Linux — это **индивидуально собранный набор**:
- Ядро Linux (сердце системы)
- Пакетный менеджер (apt, yum, pacman)
- Предустановленные программы
- Система инициализации (systemd, OpenRC)

# **Популярные дистрибутивы Linux**

### **1. Для новичков**
- **Ubuntu** – один из самых популярных дистрибутивов, основан на Debian, простая установка и удобный интерфейс (GNOME).
- **Linux Mint** – основан на Ubuntu, ещё более дружелюбен к пользователям Windows (Cinnamon/MATE/Xfce).
- **Zorin OS** – похож на Windows/macOS, подходит для перехода с других ОС.
### **2. Универсальные / для разработчиков**
- **Debian** – стабильный и надежный, основа многих других дистрибутивов.
- **Fedora** – использует новейшие технологии (поддерживается Red Hat), часто выбирают разработчики.
- **openSUSE** – мощный дистрибутив с инструментами для администрирования (Tumbleweed – rolling-release, Leap – стабильный).
### **3. Для продвинутых пользователей**
- **Arch Linux** – минималистичный, гибкий, rolling-release (обновления без переустановки).
- **Gentoo** – для полного контроля над системой (сборка из исходников).
- **Slackware** – один из старейших, минимализм и простота конфигурации.
    
### **4. Легковесные дистрибутивы**
- **Lubuntu** – Ubuntu с LXQt, подходит для слабых ПК.
- **Puppy Linux** – очень легкий, может работать прямо с флешки.
- **AntiX** – для очень старых компьютеров (основан на Debian).
    
### **5. Специализированные**
- **Kali Linux** – для пентеста и кибербезопасности.
- **Tails**  – работает через Tor.
- **Parrot OS** – Анонимность и пентест
- **Ubuntu Studio** – для мультимедийного производства (аудио, видео, графика).


# Многие дистрибутивы Linux основаны на других, создавая целые «семейства».

### **Debian-семейство**

- **Debian** – один из старейших и самых уважаемых дистрибутивов, известен стабильностью.
    - **Ubuntu** – самый популярный потомок Debian, разрабатывается Canonical. Добавляет удобство и частые обновления.
        - **Linux Mint** – основан на Ubuntu, но с более традиционным интерфейсом (Cinnamon/MATE).
        - **Pop!_OS** – оптимизирован для разработчиков и геймеров (от System76).
        - **Zorin OS** – «Windows-подобный» дистрибутив для новичков.
    - **Kali Linux** – специализированная версия Debian для пентеста и безопасности.

### **Примеры наслоений**
- **Kali Linux** основан на Debian, но заточен под безопасность.
- **Ubuntu** основан на Debian, но имеет свои репозитории и графическую оболочку.
- **Linux Mint** основан на Ubuntu, но меняет интерфейс и набор ПО.

### **Arch Linux-семейство**
- **Arch Linux** – rolling-release, DIY-подход (устанавливается вручную).
    - **Manjaro** – более дружелюбный Arch с графическим установщиком.
    - **EndeavourOS** – почти чистый Arch, но с удобным инсталлятором.

## **Пакетный менеджер и экосистема**

**Пакетный менеджер** — это программа, которая автоматизирует установку, обновление, настройку и удаление программного обеспечения в Linux.

Он работает с **пакетами** — архивными файлами, содержащими:
- Исполняемые файлы программы (`/usr/bin/`).
- Библиотеки (`/usr/lib/`).
- Конфигурационные файлы (`/etc/`).
- Документацию (`/usr/share/doc/`).

## **2. Основные функции пакетного менеджера**
- **Установка** программ (`install`).
- **Обновление** системы (`upgrade`).
- **Поиск** пакетов (`search`).
- **Удаление** ненужного ПО (`remove`).
- **Решение зависимостей** (автоматическая установка недостающих библиотек).

### В разных дистрибутивах используются свои форматы пакетов и менеджеры:

|Дистрибутив|Формат пакета|Менеджер|Пример команды|
|---|---|---|---|
|Debian/Ubuntu|`.deb`|`APT`|`sudo apt install firefox`|
|RHEL/Fedora|`.rpm`|`DNF` (или `YUM`)|`sudo dnf install chromium`|
|Arch Linux|`.pkg.tar.zst`|`Pacman`|`sudo pacman -S neofetch`|
|openSUSE|`.rpm`|`Zypper`|`sudo zypper install vlc`|
|Gentoo|(сборка из исходников)|`Portage`|`sudo emerge firefox`|
### **Экосистема дистрибутива это:**
Это совокупность:
- **Репозиториев** (официальных и пользовательских хранилищ пакетов).
- **Документации** (вики, форумы).
- **Инструментов** для сборки пакетов (`dpkg`, `rpmbuild`).
- **Сообщества** (разработчиков и пользователей).
    

#### Примеры экосистем:
- **Debian/Ubuntu**: Огромные репозитории (`main`, `universe`), Snap-пакеты.
- **Arch Linux**: AUR (Arch User Repository) — тысячи пользовательских пакетов.
- **Fedora**: Быстрый доступ к новым технологиям (например, Wayland по умолчанию).
