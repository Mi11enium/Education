**Recon-ng** — это мощный инструмент для разведки с открытым исходным кодом, написанный на Python. Он используется для сбора информации о цели в рамках тестирования на проникновение или анализа безопасности. Инструмент помогает находить поддомены, email-адреса, документы и данные из открытых источников.

**Recon-ng** поддерживает интеграцию с различными API, включая Shodan, Google, Bing и другие сервисы. Он имеет модульную структуру, что позволяет гибко настраивать процесс сбора данных. Все найденные данные сохраняются во встроенной базе SQLite для дальнейшего анализа.

Recon-ng — это в основном пассивный OSINT-инструмент, но с элементами активного сбора данных. Его основная работа строится на анализе открытых источников информации без прямого взаимодействия с целевой инфраструктурой. Он собирает данные через публичные API, поисковые системы, архивные сервисы и другие доступные ресурсы, что делает его пассивным средством разведки.

Однако некоторые модули Recon-ng могут выполнять действия, которые можно отнести к активному OSINT. Например, модули для брутфорса поддоменов делают прямые DNS-запросы, а интеграция с такими сервисами, как Shodan, позволяет получать информацию об открытых портах и сервисах. Это уже ближе к активным методам, так как подобные запросы могут фиксироваться целевыми системами.

Грань между пассивным и активным OSINT в Recon-ng тонкая. Если использовать только модули, работающие с кэшированными данными и публичными API, то инструмент остаётся пассивным. Но при задействовании функций, которые отправляют запросы непосредственно к целевым доменам или их поддоменам, Recon-ng начинает действовать как активный инструмент.

# Первое использование
recon-ng - запуск интерфейса
help - команда помощи

recon-ng основан на модулях. То есть, нам нужно загрузить модуль в инструмент, и каждый из модулей будет использоваться для поиска определенной информации.

modules search - отображение всех установленных модулей
marketplace search - список всех доступных к установке модулей
marketplace search whois - поиск всех модулей, в которых есть слово whois

marketplace info <путь_к_модулю> - узнать информацию о модуле
marketplace install <путь_к_модулю> - установить модуль
modules load <путь_к_модулю> - загрузить модуль
После загрузки, чтобы посмотреть команды модуля, нужно два раза нажать Tab

options list - какие данные модуль требует для ввода
options set SOURCE microsoft.com - заменяем source на сканируемый домен
run - запускаем модуль
show contacts - показать список найденных контактов
