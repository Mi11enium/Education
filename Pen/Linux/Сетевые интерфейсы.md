## **Интерфейс `lo` (Loopback, локальный)**
### **Что это?**
Виртуальный интерфейс, который есть в **любой** Linux-системе. Используется для:
- Тестирования сетевых приложений без реального подключения.
- Доступа к локальным сервисам (например, веб-серверу на `127.0.0.1`).
### **Пример вывода `ip a` для `lo`:**

1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 qdisc noqueue state UNKNOWN group default qlen 1000
    link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00
    inet 127.0.0.1/8 scope host lo
    inet6 ::1/128 scope host 

- **`127.0.0.1`** — стандартный IPv4-адрес loopback.
- **`::1`** — IPv6-аналог.
- **`mtu 65536`** — максимальный размер пакета (для локальных данных).
### **Практическое применение:**

ping 127.0.0.1          # Проверка работоспособности сети "на себе"
curl http://localhost   # Обращение к локальному веб-серверу

## **Интерфейс `eth0` (Ethernet, интернет)**
### **Что это?**
Физический или виртуальный интерфейс для подключения к сети через:
- Кабель (Ethernet).
- Виртуальные машины (например, в VirtualBox).

> **Важно:** В современных системах интерфейсы могут называться иначе (например, `enp0s3` — это новый стандарт именования).

### **Пример вывода `ip a` для `eth0`:**
2: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc fq_codel state UP group default qlen 1000
    link/ether 00:1a:2b:3c:4d:5e brd ff:ff:ff:ff:ff:ff
    inet 192.168.1.100/24 brd 192.168.1.255 scope global dynamic eth0
    inet6 fe80::21a:2bff:fe3c:4d5e/64 scope link 

- **`192.168.1.100/24`** — IPv4-адрес и маска подсети.
- **`fe80::...`** — IPv6-адрес (link-local).
- **`state UP`** — интерфейс активен.
- **`mtu 1500`** — стандартный размер пакета для Ethernet.
