**cmd** (**терминал** или **командная строка**) — это текстовый интерфейс для управления Linux без графической оболочки (GUI).  
Это текстовый портал в самое сердце операционной системы. В отличие от графического интерфейса (GUI) с его кнопочками и окошками, терминал даёт **прямой доступ к ядру Linux**.

# Преимущества терминала
1. Полный контроль над системой. Доступ к функциям, которых нет в GUI.
2. Автоматизация с помощью команд в консоли.
3. Работа на слабом железе.
4. Скрытность. Нет файлов журналов GUI. Можно очистить историю команд одной строкой.

# root и sudo
В Linux, **root** — это суперпользователь с неограниченными правами, а **sudo** — механизм временного предоставления таких прав обычным пользователям.

## root

**Учётная запись с UID=0** (идентификатор пользователя 0).
Имеет **полный контроль** над системой.
Может **удалять, изменять и читать любые файлы**, включая системные.
Опасен при неаккуратном использовании.
Что бы войти в root воспользуйтесь командой **`sudo su`** и введите пароль от вашей Kali Linux:
Чтобы выйти из режима root введите **`exit`**

## sudo
**sudo** (от _SuperUser DO_) — это:
- Способ **временно** получить права root.
- Записывает все действия в логи (`/var/log/auth.log`).
- Требует, чтобы пользователь был в группе `sudo` или `wheel`.

Чтобы использовать команду с правами root воспользуйтесь следующей командой:
**`sudo ls`**

# Основные команды терминала
## Навигация и просмотр текущего состояния
| Команда | Описание                                                                              | Пример использования              |
| ------- | ------------------------------------------------------------------------------------- | --------------------------------- |
| `pwd`   | Показать **п**уть до **т**екущей **р**абочей **д**иректории (Print Working Directory) | `pwd`                             |
| `ls`    | Вывести **с**писок файлов и папок в текущей директории                                | `ls`, `ls -la` (подробный список) |
| `cd`    | **П**ерейти в **д**ругую **д**иректорию (Change Directory)                            | `cd /home/user`                   |

## Создание файлов и директорий

| Команда | Описание                                                       | Пример использования |
| ------- | -------------------------------------------------------------- | -------------------- |
| `mkdir` | **Создать** новую **директорию** (MaKe DIRectory)              | `mkdir NewFolder`    |
| `touch` | Создать новый **пустой файл** или обновить время его изменения | `touch file.txt`     |
## Копирование, перемещение и переименование
| Команда | Описание                                                        | Пример использования                  |
| ------- | --------------------------------------------------------------- | ------------------------------------- |
| `cp`    | **К**о**пи**ровать файл или директорию (CoPy)                   | `cp file.txt /backup/`                |
| `mv`    | **Переместить** файл/директорию или **переименовать** их (MoVe) | `mv old.txt new.txt` (переименование) |

## Удаление файлов и директорий
**Внимание!** Команды удаления в Linux, особенно `rm`, очень мощные и необратимые. Удалённые файлы обычно нельзя восстановить.

| Команда | Описание                                                                 | Пример использования    |
| ------- | ------------------------------------------------------------------------ | ----------------------- |
| `rm`    | **У**далить **ф**айл (ReMove)                                            | `rm file.txt`           |
| `rmdir` | **Удалить пустую** директорию (ReMove DIRectory)                         | `rmdir EmptyFolder`     |
| `rm -r` | **Рекурсивно удалить** директорию со всем её содержимым (**-r**ecursive) | `rm -r FolderWithFiles` |
## Просмотр и анализ содержимого файлов
Команды для чтения и проверки содержимого файлов без их редактирования.

| Команда | Описание                                                                        | Пример использования                                        |
| ------- | ------------------------------------------------------------------------------- | ----------------------------------------------------------- |
| `cat`   | Вывести всё **содержимое файла** в терминал (con**cat**enate)                   | `cat file.txt`                                              |
| `less`  | Просмотреть файл с **возможностью прокрутки** (более удобен для больших файлов) | `less large_file.log`                                       |
| `head`  | Вывести **первые** 10 строк файла                                               | `head file.log`                                             |
| `tail`  | Вывести **последние** 10 строк файла. Ключ `-f` полезен для мониторинга логов.  | `tail log.txt`, `tail -f log.txt` (следить за обновлениями) |
## Поиск и вспомогательные команды

# Управляющие команды

Иногда очень полезно одновременно выполнять команды.
Рассмотрим основной способ параллельного запуска команд в Linux/bash:

### 1. Последовательное выполнение (с условиями)
- **`&&` (Логическое И)**  
    Команда после `&&` выполнится **только если** первая команда завершилась успешно.
    command1 && command2  # command2 запустится, только если command1 успешно выполнена
- _Пример:_  
    `make && make install` — установка произойдёт только при успешной сборке.
    
- **`||` (Логическое ИЛИ)**  
    Команда после `||` выполнится **только если** первая команда завершилась с ошибкой.
    command1 || command2  # command2 запустится, только если command1 завершилась с ошибкой
- _Пример:_  
    `ping -c1 host || echo "Хост недоступен"` — вывод сообщения при неудачном ping.
    
### 2. Параллельный запуск

- **`&` (Фоновое выполнение)**  
    Команда завершается символом `&`, что позволяет запускать её в фоновом режиме. Несколько команд, разделённых `&`, выполняются параллельно:
    command1 & command2 &  # command1 и command2 запускаются одновременно
- _Пример:_  
    `sleep 10 & sleep 5 &` — оба процесса `sleep` работают параллельно.
    - **Управление параллельными задачами**
        - `wait` — ожидание завершения всех фоновых процессов.
		- `jobs` — просмотр списка фоновых задач.
	    - `fg %n` — переход к задаче `n` в foreground.
        - `kill %n` — завершение задачи `n`.
### 3. Группировка команд

- **`{ }`** — выполнение команд в текущей оболочке:
{ command1; command2; }  # Фигурные скобки должны отделяться пробелами

- **`( )`** — выполнение в под-оболочке (изолированно):
(command1 && command2) || command3  # Группировка для управления логикой

## Справочные команды
| Команда                           | Описание                                             | Пример использования    |
| --------------------------------- | ---------------------------------------------------- | ----------------------- |
| `help`                            | Показать список **встроенных команд** shell          | `help`                  |
| `help command_name`               | Показать справку о **конкретной встроенной команде** | `help cd`, `help exit`  |
| `type command_name`               | Показать тип команды (встроенная или внешняя)        | `type cd`, `type ls`    |
| `command -h` или `command --help` | Краткая справка по **внешней команде**               | `ls --help`, `mkdir -h` |
| `man command_name`                | Открыть **руководство** по команде                   | `man ls`, `man mkdir`   |
| `man -k keyword`                  | **Найти** руководства по ключевому слову             | `man -k "copy file"`    |
| `whatis command_name`             | Краткое описание команды                             | `whatis ls`             |
| `apropos keyword`                 | Аналог `man -k` - поиск по ключевым словам           | `apropos network`       |
