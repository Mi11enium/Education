

По объекту тестирования

- функциональное
- производительности (количественные хар-ки)
   - нагрузочное (тест компонента под постоянной нагрузкой)
   - стресс (тест на максимальную нагрузку в единицу времени)
   - стабильности (тест на разные нагрузки на продолжительном времени)  
- конфигурационное (оборудование, ОС, различные настройки)
   - совместимости (ОС, оборудование)
- UI тестирование (интерфейса)
  - UX - юзабилити
- Security / Compliance (проверка соответствия требованиям регуляторов)
  - локализации (корректный перевод, валюты )

Модульное тестирование (unit-testing)
Выполняется разработчиками или их коллегами.
На соответствие изначально задуманному поведению тестируется минимальный
компонент (unit) системы (программа, функция, процедура)
Часто используются фреймворки, заглушки, драйвера и фейковые объекты (mock)

Интеграционное тестирование (integration testing)
индивидуальные модули (unit'ы) объединяются и тестируются вместе
Основная задача - выявление ошибок при взаимодействии модулей
Часто используются фреймворки, заглушки, драйвера и фейковые объекты (mock)

Системное тестирование (System testing)
Тестируется полностью собранная система
Основная задача - определение соответствия требованиям

Приемочное тестирование End-to-End
Тестируется полностью собранная система + все взаимосвязанные системы, через которые проходят принимаемые и отправляемые данные.
Полный набор элементов для тестирования определяется вместе с заказчиком.
